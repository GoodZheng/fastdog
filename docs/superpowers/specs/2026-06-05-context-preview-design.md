# 上下文预览面板设计

## 概述

在搜索结果下方区域，将匹配行列表与文件预览左右拆分。点击匹配行时，右侧 AvalonEdit 控件显示整个文件内容并滚动到对应位置，所有匹配文本用黄色背景高亮标记。

## 布局

```
┌─────────────────────────────────────────────┐
│  搜索条件区（不变）                            │
├─────────────────────────────────────────────┤
│  文件列表 DataGrid（不变）                     │
├──────────────────┬──────────────────────────┤
│  匹配行列表       │  文件预览 (AvalonEdit)      │
│  (现有 ListBox)   │  - 显示整个文件内容          │
│  点击某行 →       │  - 行号 + 语法高亮           │
│                  │  - 所有匹配文本黄色高亮       │
│                  │  - 自动滚动到选中行           │
├──────────────────┴──────────────────────────┤
│  状态栏（不变）                               │
└─────────────────────────────────────────────┘
```

- 匹配行列表和文件预览之间用 GridSplitter 分隔，可拖拽调整宽度
- 匹配行列表默认占 35%，预览区占 65%
- 选中文件后，预览区立即加载文件内容并高亮所有匹配位置
- 点击匹配行时，预览区滚动到对应行

## 数据流与交互

### 选中文件时

1. 用户点击文件列表中的文件 → `SelectedResult` 变更
2. ViewModel 读取该文件内容，设置到 `FileContent` 属性
3. AvalonEdit 加载文件内容，根据文件扩展名自动选择语法高亮规则
4. 遍历该文件的所有 `MatchLine`，用 `TextMarker` 在所有匹配位置添加黄色背景高亮
5. 如果有匹配行，自动选中第一个匹配行并滚动到该位置

### 点击匹配行时

1. 用户点击左侧匹配行 → `SelectedMatchLine` 变更
2. ViewModel 通知预览区滚动到 `SelectedMatchLine.LineNumber`
3. AvalonEdit 调用 `ScrollTo(line, 1)` 跳转

### 切换文件时

1. 清除旧的 `TextMarker` 高亮
2. 加载新文件内容，重新标记匹配位置

### 新增 ViewModel 属性

- `string FileContent`：当前选中文件的完整内容
- `OnSelectedResultChanged` 中直接处理文件加载和标记
- `OnSelectedMatchLineChanged`：监听匹配行切换，触发滚动

## 语法高亮与匹配标记

### 语法高亮

- 使用 AvalonEdit 内置的 `HighlightingManager.Instance`，根据文件扩展名自动匹配
- 常见格式开箱即用：`.cs`, `.xaml`, `.xml`, `.json`, `.py`, `.js`, `.ts`, `.java`, `.cpp`, `.md` 等
- 不识别的扩展名默认纯文本显示

### 匹配文本高亮

- 使用 AvalonEdit 的 `ITextMarker` 机制（通过 `TextMarkerService`）
- 每个匹配位置根据 `MatchLine.MatchStart` 和 `MatchEnd` 计算偏移量，创建黄色背景标记
- `MatchStart/MatchEnd` 是行内偏移，需转换为文档全局偏移（累加前面所有行的长度）
- 所有匹配标记统一黄色半透明背景（`#FFFF00`），不区分当前选中行

### 标记生命周期

- 切换文件时清除所有旧标记，重新创建新文件的标记
- 搜索完成后，如果当前文件在新结果中，重新加载并标记

## 边界情况与错误处理

### 大文件

- 文件超过 5MB 时，预览区显示前 5000 行并提示"文件过大，仅显示部分内容"
- 匹配标记也只标记显示范围内的部分

### 二进制文件

- 检测到二进制文件（如 `.exe`, `.dll`, `.png` 等）时，预览区显示"二进制文件，无法预览"
- 已知二进制扩展名列表做初步判断

### 文件不可读

- 文件被占用或无权限时，预览区显示错误信息，不弹窗

### 编码问题

- 优先用 AvalonEdit 的 `EncodingDetector` 自动检测
- 检测失败时回退到 UTF-8

### 文件已删除/移动

- 搜索结果引用的文件在预览时可能已不存在，显示"文件未找到"

## 技术选型

- **ICSharpCode.AvalonEdit** NuGet 包（方案 C）
- 支持行号显示、语法高亮、文本标记、滚动定位
- 后续可扩展：代码折叠、搜索面板等
